# ① 課題名

TriArk チャット — 陸海空を繋ぐ旅のプラットフォームの予約チャット

## ② 課題内容（どんな作品か）

- Firebase（Authentication + Firestore）で作った、ログイン認証付きのリアルタイムチャットアプリ
- 私が構想している旅行プラットフォーム「TriArk」の、ゲスト⇄ホスト間メッセージ機能を想定して制作
- 単なるチャットではなく、Airbnb のような「予約プラットフォームのチャット」の仕組みを再現することを目標にした

## ④ アプリのログイン用IDまたはPassword

- メール／パスワードで新規登録すれば、どなたでも利用可能
- 動作確認用の例 — ゲスト: guest@test.com / host@test.com、PW: 123456
- 登録時に「ゲスト」「ホスト」の役割を選択できる

## ⑤ 工夫した点・こだわった点

- 役割（ゲスト／ホスト）を登録時に分け、それに応じて表示や操作を出し分けた
- 予約ステータス（問い合わせ中 → 承認待ち → 予約確定）を可視化し、ゲストがリクエスト・ホストが承認する2段階フローを実装
- ステータスが変わると「予約をリクエストしました」等のシステムメッセージが自動で会話に挿入される（Airbnb の体験を再現）
- 既読表示を、各メッセージではなく「相手の最終既読時刻」をルームに記録する軽い方式で実装
- 名前の頭文字を丸いアイコンにし、役割で色分け（ゲスト=シアン、ホスト=オレンジ）
- ブランドロゴ（陸・海・空の三つの弧と生命の樹のT）と、それに合わせた配色でデザインを統一
- Firestore のクォータを無駄に消費しないよう、既読更新の頻度を制限するなど書き込み回数に配慮

## ⑥ 難しかった点・次回トライしたいこと

- 既読更新の処理が無限ループになり、Firestore のクォータを使い切ってしまった。原因を特定して書き込み制御を入れて解決した
- AI翻訳機能をブラウザから直接 APIに繋ごうとしたが、セキュリティ制限（CORS）で動かなかった。
- 次回は、サーバー（バックエンド）を学んで AI翻訳（日本語↔英語）を本実装したい。TriArk は国境を越えたコミュニティを目指すので、翻訳は中核機能になる
- ゲスト⇄ホストのペアごとに個別ルームを作る仕組みにも挑戦したい（今回は共通ルーム）

## ⑦ フリー項目（感想など）

- 「ただ動くチャット」ではなく「予約サービスとして成立するチャット」を、事業の仕組みから逆算して設計できたのが楽しかった。
-  普段ホストとしてゲストとチャットしているデータの仕組みを垣間見たことは非常に感慨深いものだった。
